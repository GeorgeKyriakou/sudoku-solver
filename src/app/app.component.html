Info

For this assignment, I create an Angular 2 client displaying results, uploading files, and general interface, and a Node JS server which recieves an unsolved sudoku puzzle and returns the solution. Part of the logic for solving the sudoku comes from the help, and permission, Carlino Gonzalez's javascript code.

Challenges

Challenges faced for this assignment, mainly concerned the back end part of it, since solving a sudoku puzzle efficiently, can become an exponentially hard task. (e.g: this example). However, a backtracking algorithm can be used in most cases that will be efficient in solving most of the puzzles, even though some puzzles can be written with resilience against this algorithm, which leads to more complex algorithms being required (BERGGREN,NILSSON 2012).

Gonzalez, implemets this backtracking algorithm very efficiently, and was kind enough to allow me to use his brainchild for my application.

Future work || Improvements

For further improvement of this assignment, I would first try implementing the puzzle generator at a different endpoint. Then, different algorithms, e.g Rule-based solver which is proven to be more efficient that the backtracking algorithm (BERGGREN,NILSSON 2012). My third improvement would be to add a database for already solved puzzles, which would possibly improve the efficiency of the whole application even further, especially for more complex puzzles. Finally, visual improvements for the Angular client would be made.
